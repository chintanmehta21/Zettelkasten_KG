# Iter-03 — Production Memory-Bounded RAG (Design Spec)

> Replaces and supersedes the prior iter-03 attempt (now archived under
> `docs/rag_eval/common/knowledge-management/iter-03-stale/`). The first
> attempt shipped the build phase but the load-bearing artifact
> (`bge-reranker-base-int8.onnx`) was never present on the droplet, so
> the running container fell back to the lazy fp32 BGE path that ate
> ~1 GB RSS per worker. 10 of 13 eval queries returned 503 from the
> memory guard. Build phase passed; eval phase failed; iter-03 was
> functionally not shipped. This spec is the second, can-be-shipped
> attempt.

**Status:** design  
**Owner:** chintanmehta21

## 1. Goal

Hold steady-state per-worker RSS ≤ ~600 MB on the existing 2 GB / 1 vCPU
DigitalOcean droplet, support multi-user concurrent access, eliminate the
host-state-drift class of bug that defeated the prior attempt, and
self-detect calibration drift so the int8 reranker stays accurate as the
corpus grows.

| Metric | Baseline (iter-03-stale post-mortem) | Target |
|---|---|---|
| Worker RSS (steady) | 1097 MB | ≤ 600 MB |
| Cgroup mem.current (idle) | 1.27 GB | ≤ 700 MB |
| Cgroup swap.current (idle) | 770 MB | ≤ 50 MB |
| `_STAGE2_SESSION` loaded | False | True (asserted) |
| 13-query eval gold@1 | 0.23 | ≥ 0.65 |
| 13-query infra failures | 10 | 0 |
| p95 latency | 60.1 s (Caddy timeout) | ≤ 1.05× iter-02 baseline |

## 2. Background — why the prior attempt failed

| Symptom | Root cause |
|---|---|
| Worker RSS = 1097 MB | `_get_stage2_session` lazy fallback loaded full fp32 BGE (~440 MB) post-fork → no COW, duplicated per worker |
| `_STAGE2_SESSION is None` at import | `models/bge-reranker-base-int8.onnx` not on droplet (Git LFS file never committed) |
| Memory-guard 503-cascade | RSS legitimately > 90% of `mem.max=1.36 GB` once both workers each loaded fp32 |
| Audit didn't catch it | Audit checked the repo; never checked the droplet's `/opt/zettelkasten/data/models/` |
| Compose change silently no-op'd | `deploy.sh` doesn't sync compose files from repo (now fixed in main) |
| Caddy 60s response_header_timeout hit q5 | Worker swap-thrashing during inference under degraded fallback |

## 3. Architecture overview

| Layer | Change |
|---|---|
| Model artifacts | Pre-quantized `bge-reranker-base-int8.onnx`, `bge_calibration_pairs.parquet`, FlashRank dir, all committed to repo (LFS for ONNX/parquet, plain for FlashRank dir) |
| Container | Dockerfile `COPY models/ /app/models/` in runtime stage. No build-time quantize. GH Actions checkout uses `lfs: true` |
| Compose | Remove `/app/models` host-volume mount. Add new `/app/runtime` writable mount for runtime logs. `/app/models` becomes effectively read-only |
| Code | `cascade.py` rips out lazy fp32 fallback; `_STAGE2_SESSION` is the only path; failure to load fails the import. `DegradationLogger(log_dir)` decouples log path from model path |
| Deploy | `deploy.sh` adds stage2-assert (exit 88) + pre-flip RAG smoke (exit 89), both fail-loud-no-rollback per the iter-03 hot-fix pattern |
| Drift | Weekly cron workflow detects corpus drift; auto-opens PR with regenerated calibration parquet + int8 ONNX; manual `python ops/scripts/refresh_calibration_and_int8.py` is the worst-case fallback |

| Path | Storage | Size | Generated by |
|---|---|---|---|
| `models/bge-reranker-base-int8.onnx` | LFS | ~110 MB | `ops/scripts/quantize_bge_int8.py` |

`.gitattributes` already declares `models/*.onnx` and `models/*.parquet`
LFS-tracked. No new LFS rules needed.

## 5. Components — Dockerfile

Single change (in runtime stage, after the existing `COPY` of the venv +
code, before `USER appuser`):

```dockerfile
# iter-03: bake all reranker model artifacts into the image so /app/models
# is read-only at runtime and the host-state-drift bug class is eliminated.
COPY --chown=appuser:appuser models/ /app/models/
```

CI-side: `.github/workflows/deploy-droplet.yml` checkout step needs
`lfs: true` so the runner pulls the LFS payload before the docker build
sees `models/`.

## 6. Components — compose

| Mount | Before | After |
|---|---|---|
| `/app/models` | `/opt/zettelkasten/data/models:/app/models:rw` | **REMOVED** — baked into image |
| `/app/runtime` | (none) | `/opt/zettelkasten/data/runtime:/app/runtime:rw` (new) |
| `/app/kg_output` | unchanged | unchanged |
| `/app/bot_data` | unchanged | unchanged |

The new `/app/runtime` mount holds `degradation_events.jsonl` and any
future per-instance writable state. The container's rootfs stays
`read_only: true`.

## 7. Components — cascade.py

| Change | Reason |
|---|---|
| Remove the entire lazy `_get_stage2_session` body that calls `_model_manager.ensure_bge_onnx_model()` | Lazy fp32 fallback is the failure path that ate 1 GB RSS in the prior attempt. Image now guarantees int8 exists. |
| `_get_stage2_session` returns `_STAGE2_SESSION`; raises `RuntimeError` if `None` | Fail-fast at first reranker call; deploy.sh stage2-assert catches it earlier still |
| Add `_BGE_INT8_PATH` startup assertion: at module import, if `_STAGE2_SESSION is None`, log `[bootstrap] FATAL: int8 BGE missing` and continue (the deploy.sh assertion catches it; we don't crash at import to keep tests/local dev runnable when the LFS artifact isn't pulled) | Tests + dev environments without LFS pull need the module to import |
| `DegradationLogger(log_dir)` — second positional param, defaults to `os.environ.get("RAG_DEGRADATION_LOG_DIR", "/app/runtime")` | Decouple write path from model dir |
| `CascadeReranker.__init__` passes `log_dir` from env to `DegradationLogger` | Wire the new param |

## 8. Components — deploy.sh

Two new assertions, both fail-loud-no-rollback (operator triages — same
pattern as the hot-fixed cgroup-assert):

| Assertion | Probe | Exit code on failure |
|---|---|---|
| `[stage2-assert]` | `docker exec zettelkasten-${IDLE} python -c "from website.features.rag_pipeline.rerank import cascade; assert cascade._STAGE2_SESSION is not None"` | 88 |
| `[rag-smoke]` | `curl -fsS -H 'Authorization: Bearer $RAG_SMOKE_TOKEN' -d '{"sandbox_id":"$KM_KASTEN_ID","content":"$Q1_TEXT","quality":"fast","stream":false,"scope_filter":{}}' http://127.0.0.1:${IDLE_PORT}/api/rag/adhoc \| jq -e '.turn.citations[0].node_id == "gh-zk-org-zk"'` | 89 |

Order in `deploy.sh`:
1. `up -d --no-deps` (idle color)
2. `healthcheck.sh` (existing)
3. **`[cgroup-assert]`** (already shipped; exit 87 on mismatch)
4. **`[stage2-assert]`** (new; exit 88)
5. **`[rag-smoke]`** (new; exit 89)
6. `Flipping Caddy upstream` (existing)

`RAG_SMOKE_TOKEN` and `KM_KASTEN_ID` and `Q1_TEXT` are static — added to
the `STATIC_BODY` block in the deploy workflow. The smoke probe queries
the gold-citation iter-03 q1 ("zk-org/zk Go + Markdown two-fact").
None of the assertions auto-rollback.

## 9. Components — drift detection + auto-refresh

| Component | Cadence | Behavior |
|---|---|---|
| `ops/scripts/check_corpus_drift.py` | on-demand + weekly | Reads Supabase chunk count, source_type distribution, embedding centroid. Compares vs `models/calibration_baseline.json`. Returns YES/NO + breached thresholds (chunk-count Δ > 10%, source-type Δ > 5pp, centroid L2 > 0.05) |
| `ops/scripts/refresh_calibration_and_int8.py` | manual + auto | Runs `build_calibration_set.py` → `quantize_bge_int8.py` → updates `calibration_baseline.json` |
| `.github/workflows/check_calibration_drift.yml` | cron `0 2 * * 0` (Sunday 02:00 UTC) | Runs check; on drift, runs refresh, commits to `auto/calibration-refresh-YYYY-MM-DD` branch, opens PR with drift report + 3-query smoke eval in PR body |
| PR review gate | per-event | Human merges. Merge → master → existing deploy workflow rebuilds + deploys |
| Worst-case manual path | per-event | `python ops/scripts/refresh_calibration_and_int8.py && git add models/ && git commit -m "chore: refresh int8 calibration"` is the one-liner fallback |

## 13. Out of scope

| Item | Why deferred |
|---|---|
| Shrinking cgroup ceiling back below 1.3 GB | Future-proof headroom; revisit after 24h observation window |
| Re-enabling RAG_FP32_VERIFY=on | Last-resort triage step only |
| Switch to a smaller base model than BGE-reranker-base | Would require new calibration + new quality regression test — separate iter |
| Auto-merge of drift-refresh PRs | Requires high confidence in eval gate; phase 2 once drift cron has track record |
| Move FlashRank model to LFS | 33 MB plain checkout is fine; LFS overhead not worth it |
